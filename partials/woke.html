The browser is already a runtime.

Stop rebuilding the web inside the web.

Modern frontend tools are powerful, but too often they begin by hiding the platform: state disappears into closures, templates disappear into compilers, events disappear into synthetic layers, and the DOM becomes an implementation detail.

data-wrapper goes the other way.

It treats HTML as the source of truth. State lives where you can see it: in data-* attributes. Bindings are attributes. Events are native browser events. Templates are real <template> tags. The DOM is not a target to be replaced — it is the interface.

Add one Web Component. Write normal HTML. Use three tiny tokens: $ for state-to-DOM, @ for events, and _ for attribute reflection. Inspect it, style it, mutate it, debug it, and understand it with the tools the browser already gives you.

The framework is the browser.

data-wrapper is not trying to replace React, Vue, Svelte, or your application architecture. It is for the layer of interactivity that should have stayed close to HTML in the first place: forms, filters, menus, counters, toggles, small apps, docs, dashboards, prototypes, islands, and progressive enhancement.

No build step. No virtual DOM. No proprietary template language. No ceremony for the parts of the web that never needed ceremony.

Just HTML that wakes up.
