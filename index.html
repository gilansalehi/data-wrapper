    JS — register + compute
    
const app = document.getElementById('app');

const compute = () => {
  const todos  = app.state.todos  || [];
  const filter = app.state.filter || 'all';
  app.put('active', todos.filter(t => !t.done));
  app.put('view', todos.filter(t =>
    filter === 'all' ? true : filter === 'active' ? !t.done : t.done
  ));
};

app.register({
  'todo/add': e => {
    e.detail.preventDefault();
    const input = e.detail.target.querySelector('input[name="task"]');
    if (!input?.value.trim()) return;
    app.push('todos', { id: Date.now(), task: input.value.trim(), status: 'active', done: false });
    input.value = '';
  },
  'todo/toggle': e => {
    const id = Number(e.detail.delegateTarget?.value);
    app.put('todos', prev => (prev || []).map(t =>
      t.id === id ? { ...t, done: !t.done, status: !t.done ? 'done' : 'active' } : t
    ));
  },
  'todo/remove': e => {
    const id = Number(e.detail.delegateTarget?.value);
    app.put('todos', prev => (prev || []).filter(t => t.id !== id));
  },
  'filter/set': e => app.put('filter', e.detail.delegateTarget?.dataset.val),
});

app.addEventListener('data:sync', e => {
  if (['todos', 'filter'].includes(e.detail.key)) compute();
});

app.put('todos', [
  { id: 1, task: 'Master the DOM',    status: 'done',   done: true  },
  { id: 2, task: 'Ship data-wrapper', status: 'active', done: false },
]);





    The filter buttons are styled by a plain CSS attribute selector:
    data-wrapper[data-filter="done"] button[data-val="done"] { font-weight: bold } — no JS.

Computed state via data:sync

   No dependency tracking required — recompute whenever dependencies change.

  'first/set': e => demo.put('firstName', e.detail.target.value),
  'last/set':  e => demo.put('lastName',  e.detail.target.value),

CSS as the view layer


Because state IS data-* attributes, CSS attribute selectors react to state changes
   without any binding. The filter buttons above and the nav sidebar both use this pattern.



    CSS — no JS, no $ binding needed
    
/* Active filter button — pure CSS, reacts to data-filter attribute */
data-wrapper[data-filter="all"]    button[data-val="all"],
data-wrapper[data-filter="active"] button[data-val="active"],
data-wrapper[data-filter="done"]   button[data-val="done"] {
  font-weight: bold;
  text-decoration: underline;
}

/* Nav sidebar open state — no JS, data-nav-open attribute drives both
   backdrop visibility and dialog.open via $open binding */
#nav[data-nav-open="true"] .nav-backdrop { display: block; }
















Tokens



    
TokenDirectionWhat it doesExample

    
        

            $
            State → DOM
            Binds a state path to a DOM property. Works on any property the element exposes.
            $text="/username"
        

        

            *
            State → DOM structure
            Runs a structural directive. Use this only when data-wrapper owns the shape of the DOM.
            *list="/items"
        

        

            @
            Event → handler
            Delegates a native DOM event to a registered handler topic. One listener per event type on the wrapper.
            @click="todo/remove"

    Custom formatter registration
    
import { DW_FORMATTERS } from 'data-wrapper';

DW_FORMATTERS.set('initials', v =>
  String(v).split(' ').map(w => w[0].toUpperCase()).join('')
);

DW_FORMATTERS.set('ago', v => {
  const diff = Date.now() - new Date(v).getTime();
  const mins = Math.floor(diff / 60000);
  return mins < 1 ? 'just now' : `${mins}m ago`;
});

// Use: <span $text="/author?format=initials"></span>












State API




    
MethodDescriptionExample

    
        

            put(key, val)
            Set a state key. Pass a value or an updater function.
            app.put('count', n => n + 1)
        

        

            patch(key, obj)
            Shallow-merge into an existing object state key.
            app.patch('user', { name: 'Ali' })
        

        

            push(key, item)
            Append an item to an array state key.
            app.push('todos', { id: 1, task: '…' })
        

        

            pull(key, pred)
            Remove items from an array by id or predicate.
            app.pull('todos', t => t.done)
        

        

            register(map)
            Wire event topic strings to handler functions.
            app.register({ 'add': e => … })

DWRL Reference


Data Wrapper Resource Locator. A URL-shaped addressing scheme parsed entirely by
   the browser's native new URL(). The token determines intent; DWRL names the resource.



Path syntax



    
SyntaxResolves to

    
        
/keyWrapper-root state key

        
/user/nameNested: state.user.name

        
./keyItem-scoped (inside *list template only)

        
//other-id/keyCross-wrapper by id (in progress)

    




Query params



    
ParamEffectExample

    
        

            ?format=name
            Apply a named formatter. Chain multiple — applied left to right.
            $text="/price?format=currency"
        

        

            ?key=field
            Override the identity key for *list reconciliation (default: id).
            *list="/users?key=uuid"
        

    




Event topics


The @event value is the topic name emitted as a CustomEvent on the wrapper.
   Use slash paths as namespaces: todo/add, filter/set, modal/close.
