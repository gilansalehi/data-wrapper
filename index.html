Todos persist through localStorage using the $data-todos="localstorage://todos" protocol binding — no dw/sync wiring, no save callback. Open DevTools → Elements and watch the attributes update live.

Full HTML — markup + inline script, loaded from views/showcase/todos.html

CSS as the view layer

Because state IS data-* attributes, CSS attribute selectors react to state changes without any binding. The filter buttons above and the nav sidebar both use this pattern.

Password meter — module demo

Inline markup; handlers, validation rules, and derived state come from /controllers/password-meter.js via src="". As you type, each rule lights up when satisfied, and the input frame's strength class drives the colour.
HTML inline + JS from src 
const RULES = [
  { id: 'len',    label: 'At least 8 characters', test: p => p.length >= 8 },
  { id: 'upper',  label: 'An uppercase letter',   test: p => /[A-Z]/.test(p) },
  { id: 'lower',  label: 'A lowercase letter',    test: p => /[a-z]/.test(p) },
  { id: 'digit',  label: 'A number',              test: p => /\d/.test(p) },
  { id: 'symbol', label: 'A symbol',              test: p => /[^A-Za-z0-9]/.test(p) },
];

const STRENGTHS = ['', 'weak', 'weak', 'fair', 'good', 'strong'];

export default function init(wrapper) {
  const compute = (pw) => {
    const constraints = RULES.map(r => ({ id: r.id, label: r.label, met: r.test(pw) }));
    const score       = constraints.filter(c => c.met).length;
    wrapper.put('constraints', constraints);
    wrapper.put('strength',    pw ? STRENGTHS[score] : '');
  };

  wrapper.register({
    'pw/update': e => wrapper.put('password', e.target.value),
  });

  wrapper.addEventListener('dw/sync', e => {
    if (e.detail.key === 'password') compute(wrapper.state.password ?? '');
  });

  compute('');
}

The module demonstrates three things at once: handlers from a file (the register call), derived state (the dw/sync listener recomputes constraints and strength), and *list + $class rendering each rule's met/unmet state without per-item listeners.

import { DW_FORMATTERS } from 'data-wrapper';

DW_FORMATTERS.set('initials', v =>
  String(v).split(' ').map(w => w[0].toUpperCase()).join('')
);

DW_FORMATTERS.set('ago', v => {
  const diff = Date.now() - new Date(v).getTime();
  const mins = Math.floor(diff / 60000);
  return mins < 1 ? 'just now' : `${mins}m ago`;
});

State API

MethodDescriptionExample
get(path) Read a leaf at any depth. Returns undefined for missing paths. app.get('user/name')
put(path, val) Set a leaf at any depth. Pass a value or an updater function. app.put('user/name', 'Ali')
patch(path, obj) Shallow-merge into the object at path. app.patch('user', { name: 'Ali' })
push(path, item) Append an item to the array at path. app.push('todos', { id: 1, task: '…' })
pull(path, pred) Remove items from the array at path by id or predicate. app.pull('todos', t => t.done)
register(map) Wire event topic strings to handler functions. app.register({ 'add': e => … })

path is a slash-separated pURL path. 'name' reads/writes a top-level state key; 'user/name/first' drills into nested state under the user root key. Single-segment paths and nested paths share the same call sites — flat keys are the degenerate case.

DWRL Reference

Data Wrapper Resource Locator. A URL-shaped addressing scheme parsed entirely by the browser's native new URL(). The token determines intent; DWRL names the resource.

Event topics

The @event value's path becomes the topic name. The framework dispatches the CustomEvent from the actionTarget (the element carrying the @), and native bubbling carries it up to the wrapper where register() handlers fire. Use slash paths as namespaces: todo/add, filter/set, modal/close.

Configuration

Custom directives

DW_DIRECTIVES.set() 
import { DW_DIRECTIVES } from 'data-wrapper';

// A directive is a compiled handler. It runs once at wake time and returns
// a subscriber that the wrapper calls whenever its bound state changes.
DW_DIRECTIVES.set('show', ({ el }) => value => {
  el.hidden = !value;
});

// Use: <div *show="/loggedIn">…</div>
