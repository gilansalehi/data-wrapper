data-wrapper

Zero-dependency HTML-first reactivity. A framework of the browser, for the browser.

<script src="https://unpkg.com/data-wrapper/dist/data-wrapper.min.js"></script>
<!-- or: npm install data-wrapper -->

State lives in data-* attributes — inspect it live in DevTools. Three token prefixes ($ _ @) are the entire API. No build step, no virtual DOM, no proprietary template language.

Getting Started

A counter. State starts in HTML. One script block wires the button.

Open DevTools → Elements and watch data-count update live.

HTML + JS 
<data-wrapper id="counter" data-count="0">
  <button @click="count/dec">−</button>
  <span $text="/count"></span>
  <button @click="count/inc">+</button>
</data-wrapper>

<script>
  customElements.whenDefined('data-wrapper').then(() => {
    const counter = document.getElementById('counter');
    counter.register({
      'count/inc': () => counter.put('count', n => Number(n) + 1),
      'count/dec': () => counter.put('count', n => Number(n) - 1),
    });
  });
</script>

State

The data-* attributes are the state store. No separate store object, no serialisation step. Setting app.state.name = 'Ali' writes data-name="Ali" to the element. Reading app.state.todos parses JSON from data-todos.

Static binding — no JavaScript required

Initial state declared in HTML. Bindings render immediately on connect.

Bidirectional sync

Attribute mutations from any source — DevTools, external scripts, form libraries — propagate automatically via MutationObserver. Try editing data-name directly in DevTools.

Tokens

TokenDirectionWhat it doesExample
$ State → DOM Binds a state path to a DOM property. $text, $value, $checked, $href, … any property works. $text="/username"
_ State → DOM attr Binds to an HTML attribute rather than a JS property. Useful for CSS hooks via attribute selectors. (In progress.) _data-active="/isActive"
@ DOM event → handler Delegates a native event type to a registered handler. One listener per event type on the wrapper — zero per-element listeners. @click="todo/remove"

Bindings ($)

$prop="/path" maps to el[prop] = value. A handful of aliases handle the common case where HTML and JS names differ.

ShorthandMaps toUse for
$texttextContentSafe text output
$htmlinnerHTMLRich HTML — trusted sources only, XSS risk
$classclassNameDynamic class string, merged with static classes
$forhtmlFor<label> association
$readonlyreadOnlyInput read-only state

Any unlisted property falls through to el[prop] — so $value, $checked, $disabled, $href, $src, $placeholder all work as-is.

Registering a custom formatter 
import { VP_FORMATTERS } from 'data-wrapper';

// Register before mount
VP_FORMATTERS.set('initials', v =>
  String(v).split(' ').map(w => w[0].toUpperCase()).join('')
);

VP_FORMATTERS.set('pluralise', v =>
  `${v} ${Number(v) === 1 ? 'item' : 'items'}`
);

Events (@)

@event="topic/name" delegates the native event to a handler registered on the wrapper. One listener per event type on the wrapper — never per-element. Works across dynamically added children.

Handlers receive a CustomEvent. The original browser event is in e.detail:

PropertyValue
e.detailOriginal browser event (click, input, submit, …)
e.detail.targetNative event target — useful for input.value
e.detail.delegateTargetElement carrying the @ attribute

List Rendering ($list)

Bind an array to a container. The <template> child is the row definition. The reconciler diffs by item.id — only changed rows are touched. Inside the template, ./path reads from the row's item object.

State API




    
MethodDescriptionExample

    
        

            put(key, val)
            Set a state key. Accepts a value or an updater function.
            app.put('count', n => n + 1)
        

        

            patch(key, obj)
            Shallow-merge an object into an existing state key.
            app.patch('user', { name: 'Ali' })
        

        

            push(key, item)
            Append an item to an array state key.
            app.push('todos', { id: 1, task: '…' })
        

        

            pull(key, pred)
            Remove items from an array by id or predicate.
            app.pull('todos', t => t.done)
        

        

            register(map)
            Register event topic handlers on the wrapper.
            app.register({ 'todo/add': e => … })

Patterns



Computed state via data:sync


Listen for state changes and derive new values with put. No dependency tracking —
   recompute whenever dependencies change.




    
        

            
            
        

        
Full name: 

    




    JS — computed state
    
const demo = document.getElementById('computed-demo');

demo.register({
  'first/set': e => demo.put('firstName', e.detail.target.value),
  'last/set':  e => demo.put('lastName',  e.detail.target.value),
});

demo.addEventListener('data:sync', () => {
  demo.put('fullName', `${demo.state.firstName} ${demo.state.lastName}`);
});





Inline init via onload


The onload attribute fires when the wrapper connects — before any external script runs.
   Useful for self-contained components.

DWRL Reference


Data Wrapper Resource Locator. A URL-shaped addressing scheme. Parsed by the browser's native
   new URL() — no custom parser. The token ($ / _ / @)
   determines intent; DWRL names the resource.

Query params



    
ParamEffectExample

    
        

            ?format=name
            Apply named formatter. Chain multiple: applied left to right.
            $text="/price?format=currency"
        

        

            ?key=field
            Override identity key for $list reconciler (default: id).
            $list="/users?key=uuid"
        

    




Event topics


The @event attribute value is the topic name. Use slash-separated paths as a
   lightweight namespace: todo/add, filter/set, modal/open.



<button @click="todo/remove" $value="./id">Delete</button>
<form   @submit="todo/add">…</form>
<input  @input="search/update">
<button @click="filter/set" data-val="active">Active</button>

<!-- All handlers receive a CustomEvent; original event is in e.detail -->
wrapper.register({
  'todo/add':    e => { e.detail.preventDefault(); … },
  'filter/set':  e => { app.put('filter', e.detail.delegateTarget.dataset.val); },
  'search/update': e => { app.put('query', e.detail.target.value); },
});

Custom formatters



    Registering formatters
    
import { VP_FORMATTERS } from 'data-wrapper';

VP_FORMATTERS.set('initials', v =>
  String(v).split(' ').map(w => w[0].toUpperCase()).join('')
);

VP_FORMATTERS.set('pluralise', v =>
  `${v} ${Number(v) === 1 ? 'item' : 'items'}`
);

VP_FORMATTERS.set('ago', v => {
  const diff = Date.now() - new Date(v as string).getTime();
  const mins = Math.floor(diff / 60000);
  return mins < 1 ? 'just now' : `${mins}m ago`;
});
