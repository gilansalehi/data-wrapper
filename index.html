Todos app

A complete CRUD app. State in data-* attributes, actions registered in JS, rows rendered via *list, and per-row badges controlled by *if. Todos persist through localStorage, then render through wrapper state. Open DevTools → Elements and watch the attributes update live.

Live state — inspect data-todos in DevTools or expand here
Full HTML 
const app = document.getElementById('app');
const TODOS_STORAGE = 'data-wrapper:todos';
const SEED_TODOS = [
  { id: 1, task: 'Master the DOM',    status: 'done',   done: true  },
  { id: 2, task: 'Ship data-wrapper', status: 'active', done: false },
  { id: 3, task: 'Write the docs',    status: 'active', done: false },
];

const loadTodos = () => {
  try { return JSON.parse(localStorage.getItem(TODOS_STORAGE)) || SEED_TODOS; }
  catch { return SEED_TODOS; }
};

const compute = () => {
  const todos  = app.state.todos  || [];
  const filter = app.state.filter || 'all';
  app.put('active', todos.filter(t => !t.done));
  app.put('view', todos.filter(t =>
    filter === 'all' ? true : filter === 'active' ? !t.done : t.done
  ));
};

app.register({
  'todo/add': e => {
    e.detail.preventDefault();
    const input = e.detail.target.querySelector('input[name="task"]');
    if (!input?.value.trim()) return;
    app.push('todos', { id: Date.now(), task: input.value.trim(), status: 'active', done: false });
    input.value = '';
  },
  'todo/toggle': e => {
    const id = Number(e.detail.delegateTarget?.value);
    app.put('todos', prev => (prev || []).map(t =>
      t.id === id ? { ...t, done: !t.done, status: !t.done ? 'done' : 'active' } : t
    ));
  },
  'todo/remove': e => {
    const id = Number(e.detail.delegateTarget?.value);
    app.put('todos', prev => (prev || []).filter(t => t.id !== id));
  },
  'filter/set': e => app.put('filter', e.detail.delegateTarget?.dataset.val),
});

app.addEventListener('data:sync', e => {
  if (e.detail.key === 'todos') {
    localStorage.setItem(TODOS_STORAGE, JSON.stringify(app.state.todos || []));
    compute();
  }
  if (e.detail.key === 'filter') compute();
});

app.put('todos', loadTodos());

Todo data is loaded from localStorage["data-wrapper:todos"] on startup and saved when todos changes. The filter buttons are styled by a plain CSS attribute selector: data-wrapper[data-filter="done"] button[data-val="done"] { font-weight: bold } — no JS.

Password meter — module demo

Inline markup; handlers, validation rules, and derived state come from /controllers/password-meter.js via src="". As you type, each rule lights up when satisfied, and the input frame's strength class drives the colour.
HTML inline + JS from src 
const RULES = [
  { id: 'len',    label: 'At least 8 characters', test: p => p.length >= 8 },
  { id: 'upper',  label: 'An uppercase letter',   test: p => /[A-Z]/.test(p) },
  { id: 'lower',  label: 'A lowercase letter',    test: p => /[a-z]/.test(p) },
  { id: 'digit',  label: 'A number',              test: p => /\d/.test(p) },
  { id: 'symbol', label: 'A symbol',              test: p => /[^A-Za-z0-9]/.test(p) },
];

const STRENGTHS = ['', 'weak', 'weak', 'fair', 'good', 'strong'];

export default function init(wrapper) {
  const compute = (pw) => {
    const constraints = RULES.map(r => ({ id: r.id, label: r.label, met: r.test(pw) }));
    const score       = constraints.filter(c => c.met).length;
    wrapper.put('constraints', constraints);
    wrapper.put('strength',    pw ? STRENGTHS[score] : '');
  };

  wrapper.register({
    'pw/update': e => wrapper.put('password', e.detail.target.value),
  });

  wrapper.addEventListener('data:sync', e => {
    if (e.detail.key === 'password') compute(wrapper.state.password ?? '');
  });

  compute('');
}

The module demonstrates three things at once: handlers from a file (the register call), derived state (the data:sync listener recomputes constraints and strength), and *list + $class rendering each rule's met/unmet state without per-item listeners.

Tokens

TokenDirectionWhat it doesExample
$ State → DOM Binds a state path to a DOM property. Works on any property the element exposes. $text="/username"
* State → DOM structure Runs a structural directive. Built-ins are *if and *list. *if="/ready", *list="/items"
@ Event → handler Delegates a native DOM event to a registered handler topic. One listener per event type on the wrapper. @click="todo/remove"

Bindings ($)

$prop="/path" sets el[prop] = value. A handful of aliases cover the HTML/JS naming gap.

ShorthandMaps toNotes
$texttextContentSafe text output
$htmlinnerHTML⚠ XSS risk — trusted sources only
$classclassNameDynamic class string
$forhtmlFor<label> association
$readonlyreadOnly
$tabindextabIndex
$maxlengthmaxLength

Any unlisted name maps directly: $value, $checked, $disabled, $href, $src, $open, $placeholder all work as-is. The nav sidebar uses $open="/navOpen" to control the <dialog> element.

Custom formatter registration 
import { DW_FORMATTERS } from 'data-wrapper';

DW_FORMATTERS.set('initials', v =>
  String(v).split(' ').map(w => w[0].toUpperCase()).join('')
);

DW_FORMATTERS.set('ago', v => {
  const diff = Date.now() - new Date(v).getTime();
  const mins = Math.floor(diff / 60000);
  return mins < 1 ? 'just now' : `${mins}m ago`;
});

// Use: <span $text="/author?format=initials"></span>

State API

MethodDescriptionExample
put(key, val) Set a state key. Pass a value or an updater function. app.put('count', n => n + 1)
patch(key, obj) Shallow-merge into an existing object state key. app.patch('user', { name: 'Ali' })
push(key, item) Append an item to an array state key. app.push('todos', { id: 1, task: '…' })
pull(key, pred) Remove items from an array by id or predicate. app.pull('todos', t => t.done)
register(map) Wire event topic strings to handler functions. app.register({ 'add': e => … })

System events

EventTargetWhen
data:syncWrapper elementAny put/patch/push/pull. e.detail.key = changed key.
loadWrapper elementWrapper connected — triggers onload="" attribute. e.detail = the element.
data:loadWrapper elementFires after src="" resolves. e.detail.src = resolved URL.

DWRL Reference

Data Wrapper Resource Locator. A URL-shaped addressing scheme parsed entirely by the browser's native new URL(). The token determines intent; DWRL names the resource.

Path syntax

SyntaxResolves to
/keyWrapper-root state key
/firstNameCamel-cased data-first-name / state.firstName
./keyItem-scoped (inside *list template only)
//other-id/keyCross-wrapper paths are reserved for a future release

Query params

ParamEffectExample
?format=name Apply a named formatter. Chain multiple — applied left to right. $text="/price?format=currency"
?key=field Override the identity key for *list reconciliation (default: id). *list="/users?key=uuid"

Event topics

The @event value is the topic name emitted as a CustomEvent on the wrapper. Use slash paths as namespaces: todo/add, filter/set, modal/close.

Configuration

Custom directives

DW_DIRECTIVES.set() 
import { DW_DIRECTIVES } from 'data-wrapper';

// A directive is a compiled handler. It runs once at wake time and returns
// a subscriber that the wrapper calls whenever its bound state changes.
DW_DIRECTIVES.set('show', ({ el }) => value => {
  el.hidden = !value;
});

// Use: <div *show="/loggedIn">…</div>
