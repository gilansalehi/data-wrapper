Wrapper contract

The shape every <data-wrapper> presents to the framework. state reads and writes through a Proxy backed by data-* attributes; _subs holds path subscribers; _listCache holds row caches per list element; _watch and _routeEvent are the registration seams used by wire().

Tokens

The wire dispatch. $prop binds state to a DOM property; *directive invokes a registered structural directive; @event delegates a native DOM event to a registered topic. These three are the entire surface area for wiring HTML to state.

DWRL — Data Wrapper Resource Locator

Path values like /key, ./key?format=upper, or *list="/items?key=uuid" are parsed by the browser's native new URL(). The leading slash means wrapper-root state; a leading ./ means item-scoped (inside a *list template); ?format=name chains formatters; ?key=field overrides the list identity key. formatter() compiles the format chain into a single closure at wake time.

Wake

The lifecycle entry point. wake(root) walks the subtree with a TreeWalker, skipping nested wrappers, templates, and SVG (per NO_WAKE). Each wired element is tagged with _live to make re-entry idempotent. Walking happens once; every attribute is compiled into a subscriber by wire() so runtime updates never re-parse anything.

Loading via src=""

A .js or .mjs source is dynamically imported and its default export is invoked with the wrapper — useful for handler registration in a separate file. Anything else is fetched as text, replaces innerHTML, and re-wakes. After either path, data:load fires on the wrapper with the resolved URL.

State API

State proxy

State is data-*. The Proxy serializes objects as JSON on write, parses them back on read, and broadcasts subscribers on every set. A MutationObserver catches external attribute changes (DevTools edits, third-party scripts) and broadcasts those too. _isSyncing suppresses the echo from the wrapper's own writes so the same set never fires twice.

Subscriptions

The framework's update primitive. watch registers a subscriber and runs it once for initial render; broadcast calls every stored subscriber with a new value. Wrapper-scoped subscribers live in _subs[path]; row-scoped subscribers live in row.subs. Everything else — bindings, directives, list rows — composes from this primitive.

Bindings ($)

$prop="/path" sets el[prop] = value on update. Most names map directly. The table below covers the HTML-attribute / DOM-property naming gap (class → className, for → htmlFor, etc.). $class additionally preserves the element's static base classes and merges the dynamic value on top.

*list

Reconciles a child <template> against an array. Reuses rows by identity (default key id; override with ?key=field), broadcasts updated values into existing row subscribers, and renders a data-empty template when the array is empty.

Reconciliation

How *list stays cheap. reconcile() walks incoming items, reuses cached rows by identity key, broadcasts the updated item into the row's subscribers (no DOM rebuild for unchanged rows), removes rows whose ids dropped out, and appends new rows through a DocumentFragment before waking them. The DOM node is rendered output; the row record is the framework state.

*if

Toggles an element's presence in the DOM. When false, the element is replaced with a comment anchor; when true, it returns to its place and re-wakes. Useful for conditionally rendered fragments inside list rows or wrapper roots.

Event routing (@)

One delegated listener per event type per wrapper. Native browser bubbling carries the event up to the wrapper; the listener filters out events owned by nested wrappers, then emits the topic named on the declaring element. Adding handlers to dynamically-inserted DOM costs nothing — the listener is already installed.

Empty-state templates

Built-in templates referenced by data-empty="name" on a *list element. Override by declaring a <template id> with the same name anywhere in your document — the resolver prefers user-declared templates over built-ins.
