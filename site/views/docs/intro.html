data-wrapper

data-wrapper lets you build reactive HTML with one Web Component and plain browser modules. You add the framework script, place a <data-wrapper> element on the page, and point it at an HTML file that contains the view you want to render. 

<script src="https://unpkg.com/data-wrapper/dist/data-wrapper.min.js"></script>

After the script is loaded, any <data-wrapper> element can load a view file. The src attribute points to ordinary HTML. 

<data-wrapper src="counter.html"></data-wrapper>

The view file contains the component module and the HTML that should be rendered inside the wrapper. 

<script type="module" data-module="@view/counter">
    export let count = 0;
    export const doubled = () => count * 2;
    export function inc(event) { count += Number(event.target.value); }
</script>

<button @click="inc?prevent" value="1">+1</button>
<output $text="count"></output>
<output $text="doubled"></output>

The view can include a module script. Any named export from that script is available to the HTML below it. In this example, the $text bindings read count and doubled, while the @click binding calls inc when the button is clicked.

There is no build step, virtual DOM, JSX, or component compiler. The browser loads the HTML and the module directly. When an action changes exported state, data-wrapper re-reads the bindings that are connected to the DOM and updates the matching elements.

Most of the framework follows from that small loop: export values from a module, bind those values in HTML, and let actions update the exports. Lists, child components, shared modules, directives, and formatters build on the same idea. The rest of this guide introduces those pieces one at a time.
