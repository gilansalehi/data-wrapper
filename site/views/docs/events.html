Events

An @ binding connects a browser event to a function exported by the component module. The part after @ is the native event name, and the value is the exported function to call. 

<script type="module" data-module="@view/counter">
    export let count = 0;

    export function increment(event) {
        count += Number(event.target.value);
    }
</script>

<button @click="increment" value="1">+1</button>
<output $text="count"></output>

When the button is clicked, data-wrapper calls increment. Event-bound functions run inside the component flush boundary, so state changes are published to connected bindings after the function returns.

The function receives a CustomEvent from data-wrapper. The original browser event is available as event.detail.originalEvent. The event target is still the element where the @ binding was declared, so common reads such as event.target.value work as expected.

Modifiers

Add URL-style parameters to the right side of the binding when you need to control the native browser event before the exported function runs. Multiple modifiers can be combined with &.

ModifierEffectEquivalent to
?prevent Cancels the default action. event.preventDefault()
?stop Stops the event from bubbling. event.stopPropagation()
?immediate Stops other handlers on the same element from running. event.stopImmediatePropagation() 
<script type="module" data-module="@view/form">
    export function save(event) {
        const form = event.target;
        form.reset();
    }

    export function cancel(event) {
        event.target.form?.reset();
    }
</script>

<form @submit="save?prevent">
    <input name="title">
    <button type="submit">Save</button>
    <button @click="cancel?prevent&stop">Cancel</button>
</form>

Row event details

When an event binding fires inside a *list row, the event detail includes the nearest row item. That makes row actions such as remove, toggle, or edit straightforward. 

<script type="module" data-module="@view/todos">
    export let todos = [
        { id: 1, task: 'Review event bindings' },
        { id: 2, task: 'Use row details' },
    ];

    export function removeTodo(event) {
        const { item } = event.detail;
        todos = todos.filter(todo => todo.id !== item.id);
    }
</script>

<template *list="todos">
    <button @click="removeTodo">
        Remove <span $text="./task"></span>
    </button>
</template>

The row item is captured from the binding context where the event listener was wired. If the button is inside nested lists, event.detail.item is the nearest row item.
