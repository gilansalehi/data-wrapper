fit check

Good fit, bad fit.

Every tool has a shape. Here's an honest sketch of this one — and if you want the longer answer, the comparison page grades data-wrapper against nine other frameworks and ends with an interactive version of this checklist.

Reach for it when

  • Shipping files, not build artifacts, is a feature for this project
  • It's a content site, dashboard, or admin page with islands of interactivity — not an app-shell SPA
  • The team writes HTML, CSS, and vanilla JS — framework onboarding would be a real cost
  • Pre-1.0 churn is acceptable — a side project, prototype, or internal tool

Reach for something else when

  • Dynamic content must be server-rendered for SEO or first paint
  • You need routing, code-splitting, and an app shell
  • You need an ecosystem: component libraries, devtools, a hiring pool
  • State updates at streaming or animation rates — live grids, tickers, games
