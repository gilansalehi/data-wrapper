5 — compose views

Split bigger screens into smaller files.

When a page gets larger, add more wrappers instead of changing the model. Each view owns the markup and state it needs, and the parent can pass values through the src URL. 

<data-wrapper src="customer-card.html?customer"></data-wrapper>
<data-wrapper src="order-list.html?customer"></data-wrapper>

The first component can be tiny. The larger app is the same idea repeated with clearer boundaries.
