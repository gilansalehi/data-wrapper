04 — token deep dive

Each token has one job.

The token at the start of an attribute tells data-wrapper how to treat the binding. Start with simple reads, then add properties, repeated markup, loaded views, and event actions as the interface needs more behavior.

$ writes values

Read a binding and write it to the DOM.

  1. Write text.

    <output $text="count"></output>

  2. Write a property or attribute.

    <button $disabled="isSaving">Save</button>

  3. Format a value before it lands.

    <span $text="name?trim&case=title"></span>
<span $text="amount?currency"></span>

* renders structure

Use a directive when markup itself changes.

  1. Render fallback or conditional content.

    <template *if="isReady">
    <p>Ready.</p>
</template>

  2. Repeat a template for each item. Use path-style prefixes such as ./ for data relative to the current row.

    <template *list="todos">
    <li $text="./task"></li>
</template>

  3. Fill an outlet with another view.

    <template *src="panel">
    <p>Choose a panel.</p>
</template>

@ calls actions

Listen for browser events and call exported functions.

  1. Call a function on click.

    <button @click="increment">+</button>

  2. Use event modifiers when the browser has defaults.

    <form @submit="save?prevent">
    <button>Save</button>
</form>

  3. Handle row events inside repeated markup.

    <template *list="todos">
    <input type="checkbox"
           $checked="./done"
           @change="toggleTodo">
</template>
