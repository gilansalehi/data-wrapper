Binding contexts

Every wrapper starts a root binding context backed by its ComponentRuntime. Each *list item pushes a nested row context. Bindings are wired under the context where their DOM node wakes — which is what makes nested lists compose. 

wrapper component
  order row
    line row
      DOM binding
<template *list="orders">
    <h3 $text="./customer"></h3>

    <template *list="./lines">
        <span $text="./sku"></span>
        <span $text="./qty"></span>
    </template>
</template>

The outer ./customer reads from the order row. The inner ./sku and ./qty read from the line row. Bare names climb from the nearest row through outer rows to the wrapper's component runtime. Component actions still resolve from the wrapper module, and row events include the nearest item on event.detail.item.

Resolution rules

FormResolves to
namenearest row that owns the name, then outer rows, then per-wrapper instance binding, then named module export
./namenearest enclosing *list row's item only
/namethe wrapper's root component scope, bypassing row scopes

Parent-row references (../) and cross-wrapper hosts (//host/path) are reserved for later — see Known Limitations.
