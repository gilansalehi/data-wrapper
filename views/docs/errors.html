Error handling

data-wrapper does not hide framework errors. If a view cannot load or wire correctly, the failure is thrown so the browser console and your global error handlers can see it.

Load failures happen on an async path. A failed network request, more than one data-module script in a view, a blank or duplicate data-module value, a non-function default export, a factory that returns the wrong shape, or an unknown directive will reject the load. The custom element wraps that failure with the wrapper src, and the original error remains available as the cause. 

window.addEventListener('unhandledrejection', event => {
    console.error(event.reason);
});

window.addEventListener('error', event => {
    console.error(event.error);
});

Errors from your own factory or action code are not rewritten. If a factory throws, that error becomes the cause of the failed load. If an event action throws, the error behaves like an exception from an event handler. This keeps the stack trace focused on the code that failed. 

export default () => {
    throw new Error('Could not initialize customer card');
};

Lookup misses

A missing binding is handled differently from a broken load. If a normal $ or * binding cannot resolve its name, data-wrapper renders the literal binding text and writes a console.warn. That is usually a typo, but it is not fatal and it does not stop neighboring bindings from working. 

<output $text="customerName"></output>

Cross-wrapper misses are also non-fatal, but they do not render a static fallback. A binding such as //cart/total warns and leaves the target element untouched if the wrapper or binding cannot be found. A missing cross-wrapper child input warns and is omitted from props.

No error boundary

There is no dw/error event, wrapper error state, retry system, or error-boundary component model. The public mechanism is the platform mechanism: thrown errors, rejected promises, console warnings, and browser global error handlers. If a load fails during wake, the wrapper is left retryable instead of being marked as successfully loaded.
