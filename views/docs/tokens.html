Three tokens

Token Direction Purpose
$ binding → DOM $prop="name" sets el[prop] from a binding
* binding → layout *directive="name" runs a structural directive
@ event → action @event="name" calls a module function on event

Bindings resolve against the current binding context:

  • Bare name (view) checks the nearest row, outer rows, the per-wrapper instance, then named module exports.
  • Relative path (./done) reads only from the nearest surrounding *list item.
  • Root path (/view) reads from the wrapper's component scope and bypasses rows.
