Three tokens

A view file is regular HTML, so data-wrapper uses attributes to connect that HTML to your module exports. Each special attribute starts with one of three tokens. The token tells data-wrapper what kind of work the attribute should do.

Token Purpose How to...
$ Data Binding $prop="name" reads a binding and writes it to a DOM property or attribute.
* Directives *directive="name" passes a binding to a structural directive such as *list or *if.
@ Event Handling @event="action" listens for a browser event and calls an exported action.

The value on the right side of the attribute is a binding name. Most of the time, that name points to a value or function exported by the component module. Inside a *list, the same syntax can also read from the current row item. 

<button @click="inc" value="1">+1</button>
<output $text="count"></output>

<template *list="todos">
    <label>
        <input type="checkbox" $checked="./done">
        <span $text="./task"></span>
    </label>
</template>

Binding names resolve from the place where the DOM node is rendered. These lookup forms let you choose whether a binding should read from the nearest row, a parent row, the component root, or another wrapper.

  • view checks the nearest row, outer rows, the wrapper instance, and then named module exports.
  • ./done reads only from the nearest surrounding *list item.
  • ../order reads only from the parent row. Repeat ../ to move higher.
  • /view reads from the wrapper's component root and bypasses rows.
  • //cart/total reads from another loaded wrapper by DOM id.
