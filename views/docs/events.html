Events and action modifiers

@event="action" wires a native DOM event on the element to a function exported from the component module. When the event fires, the action runs inside a flush boundary: any state mutation it makes is published to subscribers after it returns.

Event modifiers

Append URL-style parameters to control the native event before the action runs. Multiple modifiers can be combined with &.

ModifierEffectEquivalent to
?prevent Cancels the default action. event.preventDefault()
?stop Stops the event from bubbling. event.stopPropagation()
?immediate Stops other handlers on the same element from running. event.stopImmediatePropagation() 
<!-- form that doesn't navigate, button that doesn't bubble -->
<form @submit="save?prevent">
    <button @click="cancel?prevent&stop">Cancel</button>
</form>

Row event details

When an @event binding fires inside a *list row, the dispatched CustomEvent carries the row's item: 

export function removeTodo(event) {
    const { item } = event.detail;
    todos = todos.filter(t => t.id !== item.id);
}
