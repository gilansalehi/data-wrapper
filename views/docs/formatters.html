Built-in formatters

Formatters are small value transforms that run after a binding is read and before it reaches the DOM or a directive. Add them as query params on a $ binding or a * directive source. 

<span $text="name?trim&case=title"></span>
<span $text="amount?currency=EUR"></span>

<template *list="people?unique=id&sort=name">
    <span $text="./name"></span>
</template>

Params run from left to right. In the first example, trim removes surrounding whitespace, then case=title converts the remaining text to title case. Formatter arguments are static strings, so use a computed export when a sort key, currency code, or label needs to change at runtime.

Text and fallback

NameSyntaxEffect
default?default=UnknownFallback for null, undefined, or empty string.
bool?bool=Yes:NoTruthy/falsy labels. Bare ?bool returns true/false.
case?case=upperupper, lower, title, or sentence.
trim?trimTrims leading and trailing whitespace.
truncate?truncate=80Caps text length with an ellipsis.
json?jsonDebug JSON display.

Collections

NameSyntaxEffect
count?count, ?count=words, ?count=charsCounts array/string length, words, or characters.
join?join=, Joins array values.
sort?sort=name, ?sort=-name, ?sort=name:descSorts an array copy by a static key.
unique?unique, ?unique=idDeduplicates primitives or objects by a static key.

Numbers and dates

NameSyntaxEffect
number?numberLocale number formatting.
fixed?fixed=2Fixed decimal places.
percent?percent, ?percent=1Percent formatting with optional fractional digits.
currency?currency, ?currency=EURCurrency formatting. Default is USD.
date?dateLocale date.
time?timeLocale time.
datetime?datetimeLocale date and time.

The legacy onoff formatter still exists as a compatibility alias for existing code. New examples should use bool. 

export let sortKey = 'name';
export let people = [
    { name: 'Ada', role: 'engineer' },
    { name: 'Grace', role: 'admiral' },
];

export const sortedPeople = () => {
    return [...people].sort((a, b) =>
        String(a[sortKey]).localeCompare(String(b[sortKey]))
    );
};

Custom formatters

Custom formatters can be registered at startup with DW_FORMATTERS. A formatter receives the current value and the static argument string from the query param. 

import { DW_FORMATTERS } from 'data-wrapper';

DW_FORMATTERS.set('status', (value, label = 'healthy') => {
    return value === 200 ? label : 'check';
});
<span $text="code?status=ready"></span>
