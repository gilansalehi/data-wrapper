Install & scripts

From a CDN

<script src="https://unpkg.com/data-wrapper/dist/data-wrapper.min.js"></script>

From npm

npm install data-wrapper
# or
bun add data-wrapper
import 'data-wrapper';

// or as ESM, for re-exporting action/flush/types:
import { action, flush } from 'data-wrapper';

Development scripts

bun install
bun run serve       # dev server on port 3000
bun run build       # ESM + minified IIFE in /dist
bun run typecheck   # tsc --noEmit

Project layout

src/lib/
    utils.ts       binding parser, readPath, DOM helpers
    engine.ts      binding contexts, wake/wire/bind, reconcile, *list/*if
    component.ts   ComponentRuntime, action(), flush()
    element.ts     <data-wrapper> custom element + load()
    index.ts       re-exports

Browser support

Modern evergreen browsers with native ES modules, custom elements, and import maps. The framework script accepts a data-shim-src attribute pointing at es-module-shims; the shim loads only when native resolution fails: 

<script src="/dist/data-wrapper.js" type="module"
        data-shim-src="https://ga.jspm.io/npm:es-module-shims@2/dist/es-module-shims.js"></script>

Your component code uses normal import syntax either way — the shim is invisible to authors. A CDN URL (above) is the simplest option; vendoring a copy works the same. With no shim configured, an unsupported browser fails with a clear error pointing you here.
