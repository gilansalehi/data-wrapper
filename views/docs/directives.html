Directives on <template>

*list and *if live on <template> elements — the template's body is what renders, and the template itself never appears in the document. 

<ul>
    <template *list="view">
        <li $class="./done?bool=done:active">
            <span $text="./task"></span>
            <template *if="./done">
                <strong>done</strong>
            </template>
        </li>
    </template>
</ul>

Behavior

  • *list reads an array from the current binding context, reconciles by item identity (default key id, override with ?key=field), and creates a nested binding context for each rendered item.
  • Existing items update in place; new items wake; missing items tear down.
  • *if shows its body when the bound value is truthy and removes it when falsy. The body wakes under the current context, so ./ bindings inside still see the surrounding row.

Custom directives

Register structural directives at startup with DW_DIRECTIVES. A directive receives parsed binding fields, its host element, the current binding context, a wake() helper for nested DOM, and a cleanup() hook for teardown owned by that context. 

import { DW_DIRECTIVES } from 'data-wrapper';

DW_DIRECTIVES.set('mount', ({ el, path, params, ctx, wake, cleanup }) => {
    const outlet = document.createElement('section');
    el.after(outlet);
    cleanup(() => outlet.remove());

    return value => {
        outlet.textContent = '';
        if (!value) return;

        const child = document.createElement(path || 'div');
        outlet.append(child);
        wake(child, ctx);
    };
});

The function returned by a directive is its updater; the framework calls it with the bound value immediately and again whenever that value changes. The framework owns the source subscription. Directive-owned DOM, timers, observers, or listeners should be registered with cleanup().

The supported directive exports are DW_DIRECTIVES, DirectiveContext, DirectiveHandler, DirectiveUpdater, and nearestItem(ctx) for reading the nearest row item when needed. Low-level row construction and owner subscription arrays are internal; custom directives should use wake() and cleanup() instead.

FieldPurpose
path, isRel, parent, paramsParsed binding string fields.
hostThe DOM id host for //id/path sources, otherwise empty.
protocolReserved parser field for future custom resolvers.
elThe element carrying the directive attribute.
ctxThe current binding context. Pass it to wake() to preserve scope.
wake(node, ctx)Wires directive-created DOM, including child wrappers.
cleanup(off)Registers teardown with the current context owner.
