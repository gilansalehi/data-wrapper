Module scope vs. default factory

Named exports are the component's shared state surface — one per module, regardless of how many wrappers mount it. That's the right default for things like app-wide theme or auth state.

When a view needs state unique to each mounted wrapper, the module may also export a default factory. The factory receives the wrapper's load URL, parsed query parameters, and resolved props, then returns an object that overlays the module scope. If the same name appears in both, the instance value wins. 

export const label = 'Counter';

export default ({ props }) => {
    let count = Number(props.start ?? 0);
    return {
        get count() { return count; },
        increment() { count += 1; },
    };
};
<data-wrapper src="counter.html?start=5"></data-wrapper>

Here count and increment are per-wrapper bindings, while label comes from the shared module scope. Query parameters configure each wrapper independently without creating another module instance.

Child inputs

Query entries on a child wrapper's src are resolved where that child is mounted and delivered to the child's factory as props. Resolved entries are stable function readers; unresolved entries are static strings. Inputs are not automatically exposed to the template; the factory returns the bindings it wants the template to see. 

<data-wrapper src="card.html?customer&status=orderStatus"></data-wrapper>
export default ({ props }) => ({
    customer: props.customer,
    status: props.status,
});
<h3 $text="customer/firstName"></h3>
<p $text="status"></p>

Because a resolved input is a function, the factory either returns it for the template to call — keeping it live, as with customer above — or calls it to snapshot a value. A literal input is a plain value used directly, as with props.start in the counter.

Child inputs use the same lookup forms as template bindings, including ../customer for a parent row and //cart/total for an explicit read from another loaded wrapper by id. Cross-wrapper input misses warn and are omitted rather than becoming static strings.

The ComponentContext shape

The factory's single argument is a frozen object:

FieldTypePurpose
wrapperHTMLElementThe mounted <data-wrapper> element itself.
urlURLThe full resolved src URL.
paramsURLSearchParamsParsed query parameters from the src.
propsReadonly objectInput values or function readers, plus reserved props.url, the full original src string.
